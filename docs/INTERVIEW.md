# LLM Agent & MCP 面试题（30 题）

说明：每题都包含建议追问方向与核心考察点，适合用于“系统设计 + 落地实现 + 线上事故复盘”三位一体的面试。

## 一、Agent 系统设计与能力边界（1-5）

1. 你如何定义“Agent”和“带工具的 Chatbot”的边界？给出你认可的最小 Agent 架构。
   追问：哪些组件是必须的（planner、tool router、memory、policy、executor、observer 等）？哪些可以后置？考察点：抽象能力、边界清晰、可演进的架构切分。

2. 设计一个事件驱动的 Agent 平台，使其在高并发与高噪声环境下仍可控（例如风控/告警场景）。
   追问：你会把哪些逻辑放在 deterministic 层（规则/阈值/过滤）？哪些交给 LLM？考察点：成本控制、噪声抑制、确定性与不确定性的分层。

3. 如何设计 Agent 的“失败语义”？分别讨论：工具失败、模型失败、上下文不足、外部依赖降级。
   追问：什么时候重试、什么时候降级、什么时候终止并上报？考察点：可靠性工程、故障分类与策略化处理。

4. 你如何定义并落实“结构化输出契约”（contracts）？如何保证版本演进不破坏下游？
   追问：schema_version 放哪里？breaking change 怎么做？考察点：契约优先、演进策略、兼容性治理。

5. Agent 系统的“可验证性”怎么做？什么叫“可审计、可回放、可复现”？
   追问：你会记录哪些 run metadata？如何最小化隐私风险？考察点：可观测性、可审计数据模型、复现的边界。

## 二、规划、工具调用与执行闭环（6-12）

6. 对于 multi-step 任务，你如何在“显式计划（plan）”与“隐式反应（react）”之间取舍？
   追问：你如何避免过度规划与无计划乱跑？考察点：策略选择、成本与成功率权衡。

7. 工具选择（tool routing）你会怎么做？给出一种可落地的路由策略（规则/向量检索/LLM router/混合）。
   追问：当工具数量从 10 增加到 200，你的方案怎么扩展？考察点：扩展性、检索与分类、冷启动策略。

8. 你如何处理工具 schema 与模型 function calling schema 的差异？“严格 JSON Schema”有什么坑？
   追问：遇到不严格 schema（anyOf/oneOf/nullable）模型经常填错怎么办？考察点：接口契约、schema 降级与校验/修复链路。

9. 如何设计“工具回执（receipt）”的数据结构，使得 manager/critic 能基于证据做二次决策？
   追问：receipt 至少应包含哪些字段（ok、latency、artifacts、evidence、error.code）？考察点：证据链、可审计闭环、可对齐的最小字段集。

10. 并发工具调用怎么做？如何在“并行”与“依赖顺序”之间调度？
   追问：如何设置超时、取消、幂等与回滚？考察点：并发控制、资源管理、工程化执行器设计。

11. 当模型产生了“错误工具调用”（错工具、错参数、越权调用），你如何纠正？
   追问：你会在什么层做拦截（router、policy、executor）？考察点：防御式设计、分层拦截、模型对齐手段。

12. 设计一个“工具预算”系统：限制每次 run 的工具次数、单工具成本、总耗时，并保证产出可用的降级结果。
   追问：预算用尽时输出怎么保证“可解释 + 可行动”？考察点：预算治理、降级协议、产品化思维。

## 三、记忆、RAG 与上下文工程（13-18）

13. 你如何选择“RAG vs 直接工具查询 vs 直接让模型猜”？给出可执行决策树。
   追问：对时效性强的数据（行情/风控头寸）为什么不适合 RAG？考察点：数据新鲜度、正确性、成本与延迟。

14. 解释一下向量检索的召回/精排策略：你会如何设计 embedding、chunking、top_k、rerank？
   追问：如何做离线评估（MRR、Recall@k）与线上评估（点击/采纳/纠错率）？考察点：检索工程、评估体系。

15. Agent 的“共享上下文（blackboard）”与“长时记忆（KB）”怎么区分？分别记录什么？
   追问：哪些内容必须落到可查询的结构化存储？考察点：数据分层、可追溯与隐私最小化。

16. 如何避免“提示词注入/数据投毒”通过 RAG 进入决策？你会在哪些点做防护？
   追问：文档可信度、来源签名、内容清洗、引用策略怎么做？考察点：RAG 安全、信任边界、引用治理。

17. 解释“引用治理”：如何保证关键结论必须引用 tool/rag/事件字段，否则降级或拒答？
   追问：如果引用缺失但业务强要求给建议，你怎么办？考察点：可控幻觉、产品策略、失败可用性。

18. 如何设计“回放（replay）”让同一 event_id 在不同 policy/prompt 版本下可对比？
   追问：允许哪些差异（文本不同、结论一致）？如何做 diff 忽略规则？考察点：治理回归、版本化、对比的可解释性。

## 四、可靠性、评测与可观测（19-24）

19. 如何为 Agent 系统做分层指标？分别列出：模型层、工具层、工作流层、业务层关键指标。
   追问：p95 latency、error rate、tool success、LLM token、rag hit rate 如何关联到一个 run_id？考察点：指标设计、可关联性、排障效率。

20. 你如何做端到端的“质量门禁（quality gate）”？门禁失败时如何触发 rewrite/critic 回路？
   追问：门禁规则是规则化还是 LLM-as-judge？如何防止 gate 被绕过？考察点：质量工程、不可绕过机制、可回归测试。

21. 你会如何处理“非确定性回归”？比如模型升级导致输出漂移但 tests 仍绿。
   追问：你如何构建回归集与基线对比？如何定义“可接受漂移”？考察点：回归治理、漂移检测、统计与阈值。

22. 解释“幂等与去重”在事件驱动 Agent 中的落点：消费侧、执行侧、工具侧分别怎么做？
   追问：用 event_id vs partition+offset 的取舍？考察点：事件语义、Exactly-once 的工程现实、幂等设计。

23. 设计 DLQ（死信队列）与恢复流程：哪些错误进 DLQ？如何重放/补偿/人工介入？
   追问：DLQ 里要保存什么（payload、错误码、尝试次数、上下文指针）？考察点：故障隔离、可恢复性、运维流程。

24. 你如何做“成本控制”？在不显著降低成功率的情况下，降低 token、降低工具调用、降低延迟。
   追问：你会优先做哪些优化（缓存、路由、prompt 压缩、模型分层）？考察点：工程优化路径、ROI 思维。

## 五、安全、权限与 Human-in-the-loop（25-27）

25. 设计 RBAC + 工具能力分级（read_only/side_effect/pii/admin），并说明它如何贯穿到执行层。
   追问：越权时返回什么错误结构？如何让模型学会“不该做就停止”？考察点：安全边界、可执行的权限模型。

26. 解释 HITL（人工审批）在 Agent 系统里的产品与工程形态：什么时候必须人工、如何最小化打扰？
   追问：审批请求需要携带哪些证据？如何防止“审批疲劳”导致安全失效？考察点：风险控制、证据链、流程设计。

27. 如何处理敏感数据：日志/trace/上下文存储/向量库中的 PII、密钥、合规要求？
   追问：脱敏策略、访问审计、数据保留周期怎么设计？考察点：安全合规、数据治理。

## 六、MCP（Model Context Protocol）深度题（28-30）

28. MCP 解决的核心问题是什么？它与传统 tool calling / function calling 的关系是什么？
   追问：为什么需要“协议层”？它带来的新风险是什么？考察点：协议视角、标准化价值、风险评估。

29. 解释 MCP 的核心对象与流程：Host/Client/Server，Tools/Resources/Prompts/Notifications 的生命周期。
   追问：在 stdio 与 streamable HTTP 两种传输下，安全与运维关注点分别是什么？考察点：架构理解、传输差异、工程落地。

30. 设计一个“安全的 MCP Server”：如何做鉴权、最小权限、速率限制、审计、沙箱与可观测？
   追问：如何防止 prompt injection 诱导调用危险工具？如何做工具暴露过滤与审批？考察点：安全工程、治理闭环、可运营性。
