# 重构与清理计划

为了增强项目可读性并删除不必要的内容，我制定了以下重构计划：

## 1. 清理冗余入口与抽象
- **修改** `src/riskmonitor_mcp/server.py`：移除底部的 `if __name__ == "__main__":` 启动代码，统一由 `main.py` 作为唯一入口。
- **删除** `src/riskmonitor_mcp/data_access/cache.py`：该文件定义了未被使用的 Cache 协议和 NoopCache，属于过度设计。

## 2. 合并与修复指标服务
目前存在两套指标系统（旧的 `metrics_service` 和新的 `prometheus_metrics_service`），且新的未被连接。
- **修改** `src/riskmonitor_mcp/tools/mcp_tools.py`：
    - 替换依赖：使用 `prometheus_metrics_service` 替代 `metrics_service`。
    - 接入埋点：在工具执行处调用 `prometheus_metrics_service.record_request`。
    - 更新工具：让 `get_service_metrics` 工具直接从 Prometheus 服务获取摘要数据。
- **删除** `src/riskmonitor_mcp/services/metrics_service.py`：旧服务已废弃。

## 3. 清理非必要文档
- **删除** `docs/INTERVIEW.md`：这是面试题库，对项目运行和理解代码逻辑没有帮助，属于“没有必要的东西”。

## 4. 验证
- 运行测试确保重构未破坏现有功能，特别是指标相关的测试。
